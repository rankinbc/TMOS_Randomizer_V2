# DataPointer and ObjectSet Compatibility

**Last Updated**: 2026-01-24
**Purpose**: Randomizer algorithm reference
**Sources**: ROM disassembly (verified), TMOS_Romhack1 manual testing
**Confidence**: HIGH (disassembly + empirical testing)

---

## Overview

DataPointer (WorldScreen byte 8) controls both **tile graphics** and **sprite graphics**. ObjectSet (WorldScreen byte 3) defines which enemies spawn. These must be compatible for correct rendering.

---

## DataPointer Bit Layout (VERIFIED)

```
DataPointer: [7][6][5][4][3][2][1][0]
              │  │  └──────┬──────┘
              │  │         └── Bits 0-5: CHR Bank Index (sprite/tile graphics)
              │  └── Bit 6: Bottom TileSection bank (0 or 1)
              └── Bit 7: Top TileSection bank (0 or 1)
```

### How It Works
1. **Bits 0-5** select which CHR pattern bank is loaded (via table at ROM $ED43)
2. **Bits 7-6** select which TileSection ROM bank to read tile layouts from
3. The CHR bank affects BOTH sprite AND background tile graphics

---

## DataPointer Formula

For any CHR index, there are exactly **4 valid DataPointer values**:

```
DataPointer = CHR_Index + (TileBankBits << 6)

Where TileBankBits:
  0 = 0x00: Top Bank 0, Bottom Bank 0
  1 = 0x40: Top Bank 0, Bottom Bank 1
  2 = 0x80: Top Bank 1, Bottom Bank 0
  3 = 0xC0: Top Bank 1, Bottom Bank 1
```

| Bits 7-6 | Binary Pattern | Top Tiles | Bottom Tiles | Add Value |
|----------|----------------|-----------|--------------|-----------|
| 0 | `00xxxxxx` | Bank 0 | Bank 0 | `+0x00` |
| 1 | `01xxxxxx` | Bank 0 | Bank 1 | `+0x40` |
| 2 | `10xxxxxx` | Bank 1 | Bank 0 | `+0x80` |
| 3 | `11xxxxxx` | Bank 1 | Bank 1 | `+0xC0` |

```python
def get_chr_index(datapointer: int) -> int:
    """Extract CHR bank index from DataPointer (bits 0-5)"""
    return datapointer & 0x3F

def get_tile_banks(datapointer: int) -> tuple:
    """Extract tile bank selection from DataPointer (bits 7-6)"""
    bits = (datapointer >> 6) & 0x03
    top_bank = (bits >> 1) & 1
    bot_bank = bits & 1
    return (top_bank, bot_bank)

def get_compatible_objectsets(datapointer: int) -> set:
    """Get all ObjectSets compatible with this DataPointer"""
    chr_index = get_chr_index(datapointer)
    return OBJECTSET_COMPATIBILITY.get(chr_index, {0x00})

def get_all_valid_datapointers(chr_index: int) -> list:
    """Get all 4 valid DataPointers for a CHR index"""
    return [chr_index + (i * 0x40) for i in range(4)]

def validate_compatibility(datapointer: int, objectset: int) -> bool:
    """Check if ObjectSet is valid for DataPointer"""
    compatible = get_compatible_objectsets(datapointer)
    return objectset in compatible

def can_swap_screens(screen_a_dp: int, screen_b_dp: int) -> bool:
    """Check if two screens can be swapped without corruption"""
    return get_chr_index(screen_a_dp) == get_chr_index(screen_b_dp)
```

## Randomizer Rules

### Rule 1: ObjectSet Must Match CHR Index
Only assign ObjectSets from the compatible set for the screen's CHR index.

### Rule 2: Screen Swapping Requires CHR Match
Two screens can only be safely swapped if their DataPointer bits 0-5 match.

### Rule 3: Cross-Area Enemy Placement
To place enemies from one area into another:
1. Change the target screen's DataPointer to match the source CHR index
2. Preserve bits 7-6 if you want to keep the same tile bank selection
3. Assign a compatible ObjectSet

### Rule 4: Excluded Screens
Do NOT randomize ObjectSet on:
- Enemy door screens
- Demon screens
- Wizard screens
- Screens with Event != 0x00 and != 0x08

### By Screen Type
| Range | Typical Usage |
|-------|---------------|
| 0x0E-0x10 | Overworld areas |
| 0x11-0x1C | Special overworld / bottom areas |
| 0x14-0x18 | Maze areas |
| 0x54-0x58 | Special maze (Ch5) |
| 0x91 | Towns |
| 0x96-0x98 | Special dungeon areas |
| 0xD1 | Towns (variant) |
| 0xD4-0xD8 | Dungeons |

### ObjectSet Value Patterns
| Range | Typical Content |
|-------|-----------------|
| 0x01-0x0F | Common overworld enemies |
| 0x10-0x17 | Special enemies / NPCs |
| 0x37-0x48 | Dungeon/maze enemies |
| 0x70-0x80 | Chapter-specific enemies |
| 0x83-0x86 | Town NPCs |
| 0xA0-0xB0 | Special encounters |
| 0xC0-0xCC | Late-game content |
| 0xD0-0xFA | Advanced enemies / bosses |

## Known Randomizer Exclusions

The randomizer excludes these screen types from ObjectSet shuffling:
- Enemy door screens (`isEnemyDoorScreen()`)
- Demon screens (`IsDemonScreen()`)
- Wizard screens (`IsWizardScreen()`)
- Screens with special events (Event != 0x00 and != 0x08)
