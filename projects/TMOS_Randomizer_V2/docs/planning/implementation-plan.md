# TMOS Randomizer V2 - UI Implementation Plan

**Created**: 2026-01-24
**Status**: Draft
**Purpose**: Step-by-step implementation guide for the UI

---

## Executive Summary

Build a web-based UI using **React + TypeScript + Tauri** that displays randomization plans generated by the Python backend. The UI allows users to review proposed changes before patching ROMs.

---

## Technology Decision

### Chosen Stack: React + Tauri

| Component | Technology | Why |
|-----------|------------|-----|
| Framework | React 18 | Component model, ecosystem, team familiarity |
| Language | TypeScript | Type safety matching Python dataclasses |
| Build | Vite | Fast HMR, ESBuild, simple config |
| Styling | Tailwind CSS | Rapid prototyping, utility classes |
| State | Zustand | Lightweight, TypeScript-native |
| Graph Viz | D3.js | Flexible, battle-tested |
| Desktop | Tauri | Smaller than Electron (10MB vs 150MB), Rust security |
| Backend | Python (subprocess) | Reuse existing randomizer code |

### Alternatives Considered

| Option | Pros | Cons | Decision |
|--------|------|------|----------|
| Electron | Mature, Node integration | Large bundle, memory heavy | No |
| PyQt6 | Single language, native | Steeper learning curve, less flexible graph rendering | Backup option |
| Web-only | No install needed | File access limitations, no offline | No |
| Tauri + Rust backend | Performance | Rewrite Python code in Rust | Future consideration |

### Phase 7: Backend Integration (Tauri)

**Goal**: Connect UI to Python backend

**Implementation**:
1. **Tauri Commands**: Create Rust commands that invoke Python
2. **Python CLI**: Add JSON output mode to randomizer CLI
3. **IPC Protocol**: JSON over stdout/stdin

**Rust Side** (`src-tauri/src/commands.rs`):
```rust
#[tauri::command]
async fn generate_plan(seed: u64, settings: Value) -> Result<Value, String> {
    let settings_json = serde_json::to_string(&settings)?;

    let output = Command::new("python")
        .args(["-m", "tmos_randomizer", "generate", "--json"])
        .arg("--seed").arg(seed.to_string())
        .arg("--settings").arg(&settings_json)
        .output()?;

    if !output.status.success() {
        return Err(String::from_utf8_lossy(&output.stderr).to_string());
    }

    let plan: Value = serde_json::from_slice(&output.stdout)?;
    Ok(plan)
}
```

**Python Side** (`tmos_randomizer/cli.py`):
```python
@app.command()
def generate(
    seed: int = typer.Option(0, help="Random seed (0 = random)"),
    settings: str = typer.Option("{}", help="Settings JSON"),
    json_output: bool = typer.Option(False, "--json", help="Output as JSON"),
):
    """Generate a randomization plan."""
    settings_dict = json.loads(settings)
    plan = Randomizer(seed, settings_dict).generate_plan()

    if json_output:
        print(plan.to_json())
    else:
        print_plan_summary(plan)
```

**Acceptance Criteria**:
- Load ROM button works
- Generate plan calls backend
- Plan loads into UI
- Patch ROM saves file
- Error handling for backend failures

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| D3 performance with large graphs | Medium | Limit visible nodes, use WebGL (deck.gl) if needed |
| Python backend startup time | Low | Warm up on app load, show loading indicator |
| Cross-platform file dialog issues | Low | Use Tauri's native file dialog |
| Tile rendering complexity | Medium | Start simple, add tile graphics later |
| Large ROM file handling | Low | Use streaming/chunks if needed |
